# 穗鈅助手 — Discord 使用說明

> 給被核准的 Discord 用戶看的上手指南。

---

## 你有兩個管道跟 bot 互動

### 1. 你的操作 channel（`#bot-操作-{你的名字}`）

- **用途**：日常工作（建單、查信、設提醒、文件處理）
- **特性**：其他同事可以看到你跟 bot 的對話（透明，方便互相學習）
- **你可以**：發文字、貼圖、上傳 PDF / 圖片讓 bot 解析

### 2. DM（直接私訊 bot）

- **用途**：敏感操作、個人備忘、移動中快速查詢
- **特性**：只有你跟 bot 看得到

---

## 常見功能

| 你想做什麼 | 怎麼說 | 結果 |
|-----------|-------|------|
| 建訂單 | 「幫我建單：百凌，USB 線 × 10，含稅」 | bot 會開表單讓你確認、再出 PDF |
| 從 PDF 建單 | 上傳報價單 PDF，附一句「幫我建」 | bot 解析後填好表單讓你確認 |
| 設提醒 | 「提醒我明天下午三點要開會」 | bot 顯示按鈕讓你確認時間 |
| 查提醒 | 「我的提醒」 | 列出所有未到期提醒 |
| 查信 | 「幫我看 info@ 有沒有新信」 | bot 列出近期未讀 |
| 印標籤 | 「印 USB 線的標籤 × 5」 | bot 發到標籤機列印 |
| 記住某事 | 「記住百凌的窗口是王經理」 | bot 存入長期記憶 |

## 注意事項

### 發言範圍

- ❌ **不要**在 `#bot-討論區` 對 bot 說話，它不會理你（那是人類交流 bot 經驗的地方）
- ❌ **不要**在別人的操作 channel 打擾（Discord 權限會直接擋下你）
- ✅ **自己的操作 channel** 或 **DM** 才會收到回應

### 敏感資料

- 客戶完整聯絡資訊、密碼提示、金額細節等建議改用 **DM**
- 一般建單、查信在操作 channel 沒問題（反正客戶看不到你的 Discord）

### 記憶 vs 對話歷史

- **記憶**：跨管道共享。你在 DM 讓 bot 記的事，在操作 channel 也能用。
- **對話歷史**：各自獨立。DM 跟 channel 是兩個獨立 session，bot 不會把 DM 的話題帶到 channel。

### 卡住的時候

- 發「我的提醒」「用戶列表」這類指令沒反應 → 你可能不是 admin 或沒那個權限
- bot 不回覆 → 先確認你是在操作 channel 或 DM；不在這兩個地方 bot 一律不理
- 遇到錯誤訊息 → 截圖到 `#bot-討論區` 問 admin
